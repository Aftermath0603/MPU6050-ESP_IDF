ESP32-P4驱动MPU6050与蓝牙透传系统 V1.2

功能特性  
✅ 姿态解算  
使用DMP算法解算姿态  
集成卡尔曼滤波平滑数据  
支持温漂补偿算法，大幅降低yaw角长时间漂移  

✅ 系统模式  
正常工作模式  
低功耗休眠模式，检测到长时间静止则进入休眠  
自动休眠唤醒机制，每隔2s唤醒一次  

✅ 蓝牙无线通信（BLE）  
ESP32-P4通过UART串口与ESP32-C3-supermini通信  
ESP32-C3作为BLE协处理器发送加速度和姿态数据  
支持手机BLE调试助手实时接收数据  
采用NimBLE协议栈，稳定低功耗  

✅ 串口透传  
P4 原始姿态数据 → 串口 → C3 → BLE → 手机  
波特率：115200  
数据格式：加速度xyz、姿态角Pitch、Roll、Yaw  
（这部分详见仓库中另一工程"esp32c3_ble"）  

硬件架构  
笔者主控开发板用的是树莓派ESP32-P4-ETH  
传感器部分  
MPU6050 A0有上拉电阻，默认地址是0x68，若接GND则地址为0x69  
| ESP32-P4      | 引脚定义 | MPU6050           | 引脚定义          |
| ------------- | -------- | ----------------- | ------------------|
| GPIO7         | SDA      | SDA               |                   |
| VBUS          | 5V       | VCC               |供电，有些版本是3V3 |
| GPIO8         | SCL      | SCL               |                   |
| GND           | 公共地   | GND               | 公共地            |

通信部分  
| ESP32-P4      | 引脚定义 | ESP32-C3          | 引脚定义 |
| ------------- | -------- | ----------------- | -------- |
| GPIO5        | UART2_TX | GPIO20             | UART1_RX |
| GPIO4        | UART2_RX | GPIO21             | UART1_TX |
| GND           | 公共地   | GND               | 公共地   |
